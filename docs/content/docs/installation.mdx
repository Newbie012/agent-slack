Agent Slack is published as `@eliya-oss/agent-slack`.

## Requirements

- Node.js 22 or newer
- A Slack workspace you can authorize
- Slack scopes for the data you want to read

## Install

<Tabs groupId="package-manager" items={['npm', 'pnpm', 'yarn']} persist>
  <Tab value="npm">

```bash
npm install -g @eliya-oss/agent-slack
```

  </Tab>
  <Tab value="pnpm">

```bash
pnpm add -g @eliya-oss/agent-slack
```

  </Tab>
  <Tab value="yarn">

```bash
yarn global add @eliya-oss/agent-slack
```

  </Tab>
</Tabs>

Verify the binary:

```bash
agent-slack describe --json
```

Or use the short alias:

```bash
aslk describe --json
```

## Install the skill (for agents)

If your agent supports [Agent Skills](https://skills.sh), add the bundled
`agent-slack` skill so it knows when to reach for the CLI and how to drive it:

```bash
npx skills add Newbie012/agent-slack --skill agent-slack
```

The skill teaches the read-first workflow, auth steps, and output conventions.
It complements the CLI binary above, so install both: the skill is the guidance,
the CLI is the tool.

## Shell completion

```bash
agent-slack completion zsh > ~/.zfunc/_agent-slack
agent-slack completion bash
```

## Local token storage

On **macOS**, the token secret is stored in the **Keychain** by default (encrypted
at rest); only profile metadata is written to disk, under
`~/.config/agent-slack/profiles.keychain.json`.

On **other platforms**, profiles are stored in a `0600` file:

```
~/.config/agent-slack/profiles.json
```

Override the store with `AGENT_SLACK_TOKEN_STORE`:

```bash
# Force the plaintext file store — needed on headless macOS (SSH, CI), where the
# Keychain would prompt or be unavailable.
AGENT_SLACK_TOKEN_STORE=file agent-slack auth login

# Force the Keychain (macOS only).
AGENT_SLACK_TOKEN_STORE=keychain agent-slack auth login
```

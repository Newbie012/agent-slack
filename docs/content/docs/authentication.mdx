Agent Slack uses named local profiles. The default profile name is `default`.

## Browser login

```bash
agent-slack auth login
```

Browser login uses Slack OAuth with PKCE and the bundled Agent Slack public Client ID. It requests user scopes and stores the returned token locally.

Useful options:

```bash
agent-slack auth login --profile work
agent-slack auth login --no-open
agent-slack auth login --auth-url-out /tmp/agent-slack-auth-url
agent-slack auth login --timeout-ms 180000
```

## Existing bot token

```bash
agent-slack auth login \
  --profile work \
  --token "$SLACK_BOT_TOKEN" \
  --scopes channels:read,channels:history,users:read \
  --json
```

Or pass a token through an environment variable:

```bash
AGENT_SLACK_TOKEN
SLK_TOKEN
SLACK_BOT_TOKEN
```

## Self-hosted OAuth

Use this for development or your own Slack app credentials:

```bash
agent-slack auth login \
  --oauth \
  --client-id "$SLACK_CLIENT_ID" \
  --client-secret "$SLACK_CLIENT_SECRET" \
  --scopes channels:read,channels:history \
  --json
```

OAuth credential environment variables:

```bash
AGENT_SLACK_CLIENT_ID
AGENT_SLACK_PUBLIC_CLIENT_ID
SLK_CLIENT_ID
SLACK_CLIENT_ID

AGENT_SLACK_CLIENT_SECRET
SLK_CLIENT_SECRET
SLACK_CLIENT_SECRET
```

## Profiles

```bash
agent-slack auth status --profile work --json
agent-slack auth scopes --profile work --json
agent-slack auth profiles list --json
```

Delete a profile:

```bash
agent-slack auth logout --profile work --yes --json
```

`logout` is destructive and requires `--yes`. By default it also **revokes the
token on Slack** (via `auth.revoke`) before removing the local profile, so the
credential is actually invalidated and not just forgotten. If revocation fails
(offline, or the token is already invalid) the local profile is still removed and
a warning is returned. Pass `--no-revoke` to keep the token valid on Slack and
only remove it locally:

```bash
agent-slack auth logout --profile work --yes --no-revoke --json
```

## Where tokens are stored

On **macOS**, the token secret is stored in the **Keychain** by default
(encrypted at rest); only profile metadata is written to
`~/.config/agent-slack/profiles.keychain.json`. On **other platforms**, profiles
are stored in a `0600`-permission file at `~/.config/agent-slack/profiles.json`.

Override the store with `AGENT_SLACK_TOKEN_STORE`:

```bash
# Force the file store — needed on headless macOS (SSH, CI), where the Keychain
# would prompt or be unavailable.
AGENT_SLACK_TOKEN_STORE=file agent-slack auth login

# Force the Keychain (macOS only).
AGENT_SLACK_TOKEN_STORE=keychain agent-slack auth login
```

## Token type

Commands use the bot token when present, and fall back to the user token otherwise.

Be explicit when needed:

```bash
agent-slack conversation history C123 --token user --json
agent-slack api call auth.test --token bot --json
```

Valid token types:

```
user
bot
admin
app
```
